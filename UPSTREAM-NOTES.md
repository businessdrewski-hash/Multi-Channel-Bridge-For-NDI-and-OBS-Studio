## Data flow

### Sender

1. DistroAV Main Output is configured as an OBS multi-track raw output.
2. Two selected stereo OBS mixes are copied into four-block, fixed-capacity queues.
3. Sender Sync Core 2.0 pairs compatible blocks and packs four-channel planar float audio without runtime allocation or a packing mutex.
4. DistroAV’s existing NDI sender sends the normal video frame and the four-channel audio frame.
5. Bridge-enabled audio and video receive explicit NDI timecodes derived from their OBS timestamps.

### Receiver

1. DistroAV receives the normal NDI video and raw planar audio.
2. The A/V Governor records raw NDI timing, learns a filtered baseline, and maps both paths onto a common future OBS timeline before output.
3. Audio channels 1-2 and 3-4 are exposed as two independent OBS audio-only proxy sources.
4. The original combined audio is suppressed only when both proxy outputs are active.
5. Video continues through the normal DistroAV NDI source.

## Build through GitHub

A reviewer can:

1. fork this repository;
2. enable GitHub Actions in the fork;
3. run **Build Multichannel Bridge for DistroAV (Windows)**.

The workflow downloads DistroAV 6.2.1, applies the patch, runs the standalone sender/governor tests and static safety audits, builds DistroAV, creates the installer and portable package, and uploads a generated patch for review.

## Equivalent local integration

```powershell
git clone https://github.com/<owner>/<repo>.git
cd <repo>

git clone --recursive --branch 6.2.1 https://github.com/DistroAV/DistroAV.git distroav
python hotfix/scripts/patch_distroav.py distroav

cd distroav
$env:CI = 'true'
.\.github\scripts\Build-Windows.ps1 -Target x64 -Configuration RelWithDebInfo
```

DistroAV’s normal Windows prerequisites and a supported PowerShell version are still required.

## Main review points

- `patch_main_output`: enables two selected OBS mixer tracks for Main Output.
- `patch_ndi_output`: packs the two stereo callbacks into one four-channel NDI audio frame and assigns shared OBS-derived NDI timecodes.
- `sender-sync-core.cpp`: fixed-storage pairing, canonical timestamps, discontinuity epochs, and silence fallback.
- `patch_ndi_source`: inserts receiver hooks immediately before DistroAV calls the OBS audio/video output APIs.
- `multichannel-bridge.cpp`: role UI, source attachment, channel splitting, proxy sources, suppression, diagnostics, and installer-facing configuration.
- `av-governor.cpp`: robust baseline/filtering, drift confidence, shared OBS playout mapping, frame-boundary video correction, atomic recovery, audio fades, and flight recorder.

## Known limitations

- The overlay intentionally refuses to patch a DistroAV version other than 6.2.1.
- This alpha has not yet been proven across all OBS, NDI Runtime, GPU, network, and audio-device combinations.
- The governor relies on source timestamps and OBS asynchronous-source scheduling; it is not a replacement for hardware genlock.
